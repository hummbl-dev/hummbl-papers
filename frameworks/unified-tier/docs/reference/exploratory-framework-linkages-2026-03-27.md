## Working Hypothesis

The Unified Tier Framework is more useful if it distinguishes:

1. **Problem wickedness**
- how contested, uncertain, interdependent, urgent, and governance-constrained the problem is

2. **Exploratory framework demand**
- how much model variety, operator diversity, and composition depth are needed to work the problem well

These are related but not identical.

Possible consequence:

- two problems could both score as Tier 4
- but one may need only a narrow, disciplined Base24 or Base36 path
- while another needs Base42 or BaseN-open exploratory composition

## Design Opportunity

This suggests a richer next-generation structure:

- `wickedness score`
- `exploratory demand score`
- `recommended Base profile`
- `recommended control regime`
- `recommended composition posture`

Where composition posture might be:

- fixed sequence
- bounded exploration
- open exploratory chaining
- human-guided composition

## Why This Matters

Without this distinction, the current framework risks overloading one score with too many jobs:

- problem classification
- model-count recommendation
- reasoning-style recommendation
- exploration budget recommendation

That is probably too much for a single 5-question score.

## Empirical Path

This can be tested with real tasks:

- score wickedness manually
- independently score exploratory demand
- compare outcomes across:
  - Base24
  - Base36
  - Base42
  - Base120 or BaseN-open
- record whether success came from:
  - sufficient model count
  - better composition logic
  - better human/AI control regime

## Suggested Refresh Direction

Keep the five-tier wickedness framework, but add a separate exploratory layer rather than forcing all reasoning design choices into the same scalar rubric.
