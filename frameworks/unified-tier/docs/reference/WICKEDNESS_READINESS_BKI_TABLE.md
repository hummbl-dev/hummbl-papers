# Wickedness, Readiness, and BKI Table

Status: draft

## Purpose

Provide a compact working table for the three-layer direction now emerging in the framework refresh:

- `wickedness`
- `readiness`
- `BKI`

## Layer Summary

| Layer | Core Question | What It Measures | Typical Failure Mode |
| --- | --- | --- | --- |
| `wickedness` | How hard is the problem structure itself? | contestation, uncertainty, interdependence, emergence, urgency | treating a structurally difficult problem as if it were merely complicated |
| `readiness` | Can the system actually execute change against the problem? | commitment, efficacy, authority, capacity, climate, coordination | having the right diagnosis but no operational ability to act |
| `BKI` | Can the system know together well enough to act coherently? | trust, legitimacy, translation, shared interpretation, psychological safety, knowledge flow | fragmented meaning, premature closure, or untrusted knowledge |

## Candidate Dimensions

### Wickedness

- problem-definition contestation
- solution contestation
- uncertainty / information incompleteness
- interdependence / system coupling
- emergence / learning while intervening
- urgency / irreversibility

### Readiness

- implementation priority
- implementation climate
- change commitment
- change efficacy
- governance authority
- execution capacity
- coordination capability
- adaptive capacity
- bio-cognitive operating condition

### BKI

- salience
- credibility
- legitimacy
- interpretive alignment
- psychological safety
- boundary mediation
- representation quality
- knowledge-flow quality
- shared-attention durability under load

## Design Principles

### Wickedness

- classify structure, not solution quality
- do not collapse all difficulty into one scalar if the profile matters

### Readiness

- measure as a profile, not a vibe
- include confidence in the measurement itself
- treat readiness as iterative and adjustable

### BKI

- optimize for shared usable understanding, not only more information
- preserve dissent, uncertainty, and translation across roles
- treat legitimacy and trust as infrastructure

## Signals To Monitor

### Wickedness Signals

- disagreement over the problem
- disagreement over solutions
- system coupling and spillovers
- irreversible deadlines

### Readiness Signals

- priority conflict
- execution bottlenecks
- weak ownership
- adaptation without loss of core intent
- fatigue, overload, or ergonomic strain degrading execution

### BKI Signals

- dominance or exclusion in interpretation
- low psychological safety
- poor translation across domains
- low trace usability across roles
- degraded shared knowing under sustained workload

## Working Rule

Do not use these three layers interchangeably.

- a problem can be highly wicked but handled by a high-readiness, high-BKI system
- a less wicked problem can still fail in a low-readiness, low-BKI system

That distinction is one of the main reasons to refresh the framework now.
